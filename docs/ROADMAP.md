# 开发路线

## Phase 0：项目初始化

- [x] 建立私有 GitHub 仓库
- [x] 编写 MVP PRD
- [x] 建立 SwiftUI / SwiftData 目录结构
- [x] 增加 XcodeGen 工程配置
- [x] 本地生成 Xcode 工程并完成首次编译

## Phase 1：数据基础

目标：可以创建默认 8 周计划，并在本地持久化。

- [ ] 完善 SwiftData Model
- [x] 创建默认 97 kg → 88.5 kg、56 天计划
- [x] 生成每日计划体重
- [x] 生成 7 天循环饮食模板
- [x] 生成 8 周递进训练模板
- [x] 增加当前计划编辑页面
- [ ] 增加新计划创建页面
- [ ] 增加数据迁移版本策略

验收：删除并重新安装前，App 重启后数据仍存在；首次启动可自动建立默认计划。

## Phase 2：每日记录闭环

目标：用户每天可以在 2 分钟内完成核心记录。

- [x] 今日首页
- [x] 体重与腰围输入
- [x] 睡眠和晨起精神记录
- [x] 每顿饮食状态切换
- [x] 训练状态切换
- [x] 实际步数和训练时间记录
- [x] 今日完成度
- [x] 日历状态点

验收：从首页进入后，无需超过三次点击即可完成一项记录。

## Phase 3：体型照片

- [x] PhotosPicker
- [x] 相机拍摄
- [x] 正面、侧面、背面分类
- [x] 图片压缩和本地路径管理
- [x] 删除照片时清理文件
- [x] 同角度历史照片列表
- [x] 两个日期并排对比
- [x] 标准化拍摄引导
- [x] 姿态、距离和完整入镜基础质量检查
- [x] 自动裁剪与同角度对齐

## Phase 4：趋势与每周复盘

- [x] 计划体重曲线
- [x] 实际体重曲线
- [x] 7 日平均曲线
- [x] 腰围趋势
- [x] 饮食执行率
- [x] 训练执行率
- [x] 每周自动汇总
- [x] 调整建议规则

验收：缺失某一天体重时，不以 0 参与平均计算。

## Phase 5：个人试用优化

连续使用两周，记录以下内容：

- 哪些字段从未填写
- 哪些操作超过三次点击
- 哪些提醒真正有帮助
- 饮食模板是否符合日常采购
- 训练量是否过大或过小
- 是否需要 HealthKit 自动读取步数

根据使用结果删除不必要字段，优先增加快捷录入。

## P2 实验：参数化 3D 体型追踪

只有在用户持续拍摄并频繁使用照片对比后才进入实验：

- 多角度轮廓与 Vision 3D 姿态拟合
- 参数化人体模型和时间轴变化动画
- 支持设备上的 LiDAR 深度增强
- 两日期 3D 透明叠加和变化热图

不将手机扫描描述为医学级测量，不从外形推断体脂或内部健康状态。

## 建议分支策略

- `main`：稳定代码
- `feature/<name>`：功能开发
- `fix/<name>`：问题修复

每个功能通过 Pull Request 合入 `main`，即使是个人项目，也便于保留设计和实现决策。
