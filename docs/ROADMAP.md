# 开发路线

## Phase 0：项目初始化

- [x] 建立私有 GitHub 仓库
- [x] 编写 MVP PRD
- [x] 建立 SwiftUI / SwiftData 目录结构
- [x] 增加 XcodeGen 工程配置
- [ ] 本地生成 Xcode 工程并完成首次编译

## Phase 1：数据基础

目标：可以创建默认 8 周计划，并在本地持久化。

- [ ] 完善 SwiftData Model
- [ ] 创建默认 97 kg → 88.5 kg、56 天计划
- [ ] 生成每日计划体重
- [ ] 生成 7 天循环饮食模板
- [ ] 生成 8 周递进训练模板
- [ ] 增加计划创建和编辑页面
- [ ] 增加数据迁移版本策略

验收：删除并重新安装前，App 重启后数据仍存在；首次启动可自动建立默认计划。

## Phase 2：每日记录闭环

目标：用户每天可以在 2 分钟内完成核心记录。

- [ ] 今日首页
- [ ] 体重与腰围输入
- [ ] 睡眠和晨起精神记录
- [ ] 每顿饮食状态切换
- [ ] 训练状态切换
- [ ] 实际步数和训练时间记录
- [ ] 今日完成度
- [ ] 日历状态点

验收：从首页进入后，无需超过三次点击即可完成一项记录。

## Phase 3：体型照片

- [ ] PhotosPicker
- [ ] 相机拍摄
- [ ] 正面、侧面、背面分类
- [ ] 图片压缩和本地路径管理
- [ ] 删除照片时清理文件
- [ ] 同角度历史照片列表
- [ ] 两个日期并排对比

验收：图片不会存入 SwiftData 二进制字段；数据库只保存文件引用。

## Phase 4：趋势与每周复盘

- [ ] 计划体重曲线
- [ ] 实际体重曲线
- [ ] 7 日平均曲线
- [ ] 腰围趋势
- [ ] 饮食执行率
- [ ] 训练执行率
- [ ] 每周自动汇总
- [ ] 调整建议规则

验收：缺失某一天体重时，不以 0 参与平均计算。

## Phase 5：个人试用优化

连续使用两周，记录以下内容：

- 哪些字段从未填写
- 哪些操作超过三次点击
- 哪些提醒真正有帮助
- 饮食模板是否符合日常采购
- 训练量是否过大或过小
- 是否需要 HealthKit 自动读取步数

根据使用结果删除不必要字段，优先增加快捷录入。

## 暂不开发

- AI 食物识别
- 条码营养数据库
- 社交与排行榜
- 教练端
- 订阅和支付
- Apple Watch 独立 App

## 建议分支策略

- `main`：稳定代码
- `feature/<name>`：功能开发
- `fix/<name>`：问题修复

每个功能通过 Pull Request 合入 `main`，即使是个人项目，也便于保留设计和实现决策。
