- [x] 增加完成计划庆祝动画与成果摘要
- [x] 增加暂停、恢复和放弃计划的确认与反馈动画
- [x] 支持从历史计划恢复暂停的计划
- [x] 支持确认后永久删除历史计划及关联数据

## Phase 5：个人试用优化

连续使用两周，记录以下内容：

- 哪些字段从未填写
- 哪些操作超过三次点击
- 哪些提醒真正有帮助
- 饮食模板是否符合日常采购
- 训练量是否过大或过小
- 是否需要 HealthKit 自动读取步数

根据使用结果删除不必要字段，优先增加快捷录入。

## P2：实际饮食与能量建议

按以下顺序开发，先建立可核对的数据闭环，再加入照片识别：

- [x] 建立实际餐次、实际食物条目和营养快照数据模型
- [x] 支持手动输入食物、重量、单位及每 100 g / 每份营养值
- [x] 动态计算单项、本餐、当日累计能量和三大营养素
- [x] 展示与计划餐次及每日目标的差异
- [x] 基于体重、MET 和强度生成多种等效活动区间
- [ ] 用户确认后把建议作为追加活动加入今日运动计划
- [ ] 建立饮食记录与追加活动的可追踪关联
- [ ] 支持修改或删除饮食时确认是否同步调整关联活动
- [ ] 接入食物照片识别，输出候选食物、份量区间和可信度
- [ ] 照片估算必须经过用户确认后才能保存
- [ ] 增加最近使用、收藏食物和手动营养模板

验收：营养值和运动消耗均明确标注为估算；用户可以完全忽略运动建议；追加活动不覆盖原锻炼计划；无网络或识别失败时手动记录仍可用。

## P2 实验：参数化 3D 体型追踪

只有在用户持续拍摄并频繁使用照片对比后才进入实验：

- 多角度轮廓与 Vision 3D 姿态拟合
- 参数化人体模型和时间轴变化动画
- 支持设备上的 LiDAR 深度增强
- 两日期 3D 透明叠加和变化热图

不将手机扫描描述为医学级测量，不从外形推断体脂或内部健康状态。

## 建议分支策略

- `main`：稳定代码
- `feature/<name>`：功能开发
- `fix/<name>`：问题修复

每个功能通过 Pull Request 合入 `main`，即使是个人项目，也便于保留设计和实现决策。
