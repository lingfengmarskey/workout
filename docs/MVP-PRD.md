# 减脂计划 App MVP 产品需求文档

## 1. 产品概述

- 产品名称：减脂计划（暂定）
- 首期平台：iOS
- 首期用户：开发者本人
- 产品阶段：MVP
- 默认计划周期：8 周
- 数据单位：kg、cm、kcal、g、分钟、步数

本产品用于把减脂计划、每日执行记录和每周复盘集中到一个 App 中。核心不是建立复杂营养数据库，而是降低每天执行和记录的成本。

## 2. 产品目标

用户每天打开 App 后，应能快速知道：

1. 今天应该吃什么、吃多少。
2. 今天应该完成什么锻炼和多少步数。
3. 今天是否已经记录体重和体型。
4. 当前体重是否符合计划趋势。
5. 本周饮食和训练执行得怎么样。

### 成功标准

- 每日记录可在 2 分钟内完成。
- 首页直接展示当天全部任务。
- 支持至少连续 8 周记录。
- 能查看计划体重、实际体重和 7 日平均趋势。
- 核心功能无网络可用。
- 体型照片默认仅保存在本地或个人 iCloud。

## 3. 产品原则

1. **记录最少必要信息**：不要求详细记录每种调味料或每次心率变化。
2. **计划与执行分离**：系统提前生成计划，用户主要负责确认完成情况。
3. **关注长期趋势**：重点展示 7 日平均体重和每周执行率。
4. **隐私优先**：体重、腰围和照片不得默认上传第三方服务器。
5. **个人版优先**：MVP 不做社交、排行榜、订阅和公开分享。

## 4. 首期用户与默认目标

- 男性，35 岁
- 身高 180 cm
- 起始体重 97 kg
- 8 周阶段目标 88～90 kg
- 长期目标 80 kg
- 主要方式：饮食控制、步行、有氧和全身力量训练

## 5. 核心使用流程

### 5.1 首次使用

1. 创建减脂计划。
2. 输入身高、起始体重、目标体重、开始日期和计划周期。
3. 设置每日热量、蛋白质和饮水目标。
4. 选择默认 8 周模板或手动创建。
5. 系统生成每日体重目标、饮食计划和训练计划。

### 5.2 每日使用

1. 早晨记录体重。
2. 按需记录腰围、睡眠和体型照片。
3. 查看早餐、午餐、晚餐和加餐计划。
4. 每顿饭后标记完成状态。
5. 查看并完成当天训练与步数。
6. 晚间记录实际训练时长、步数、饥饿感和身体状态。

### 5.3 每周使用

1. 查看周平均体重和本周下降幅度。
2. 查看饮食与训练执行率。
3. 查看腰围、睡眠、饥饿感和精神状态。
4. 根据规则决定是否调整下一周计划。

## 6. 信息架构

底部四个 Tab：

1. **今天**：今日体重、饮食、训练与完成度。
2. **计划**：日历、每日饮食计划和每日训练计划。
3. **进度**：体重趋势、腰围趋势、体型照片和每周复盘。
4. **设置**：当前计划、提醒、单位、导出和隐私。

### 7.2 每日体重与体型记录

记录字段：

- 日期
- 早晨体重
- 腰围
- 睡眠时长
- 晨起精神 1～5
- 正面照片
- 侧面照片
- 背面照片
- 备注

自动计算：

- 当日计划体重
- 与计划差值
- 与昨日差值
- 累计减重
- 7 日平均体重
- BMI
- 计划进度

- 调用相机或相册

- 同角度历史浏览

- 拍摄提示：相同光线、距离、衣着和站姿

### 7.3 每日饮食计划

每天包括：

- 早餐
- 午餐
- 晚餐
- 加餐

每顿字段：

- 食物名称
- 计划数量
- 单位
- 估算热量
- 估算蛋白质
- 备注
- 完成状态

- 饥饿感 1～5
- 实际饮水量
- 临时加餐
- 外食情况
- 偏离原因
- 备注

编辑能力：

- 仅修改当天
- 应用到之后相同星期
- 复制某一天到其他日期
- 替换单个食物

MVP 不接入大型营养数据库，食物数据由默认模板或用户手动填写。

### 7.4 每日锻炼计划

字段：

- 训练类型
- 力量训练内容
- 有氧内容
- 热身
- 拉伸
- 计划时长
- 目标步数
- 目标强度
- 注意事项

动作字段：

- 动作名称
- 组数
- 次数或时长
- 左右侧
- 休息时间
- 使用重量
- 备注

执行记录：

- 完成
- 部分完成
- 未完成
- 主动休息
- 实际训练时长
- 实际步数
- 疲劳程度 1～5
- 疼痛部位
- 备注

安全提醒：记录胸痛、明显头晕、异常气短或严重关节疼痛时，提示停止训练并寻求医疗帮助。App 不提供医疗诊断。

### 7.5 进度趋势

体重图表展示：

- 计划体重
- 每日实际体重
- 7 日平均体重

时间范围：

- 最近 7 天
- 最近 30 天
- 当前计划
- 全部记录

其他趋势：

- 腰围
- 每周饮食执行率
- 每周训练执行率
- 平均睡眠
- 平均饥饿感
- 平均步数

### 7.6 每周复盘

每完成 7 天自动生成：

- 周平均体重
- 周末体重
- 本周下降
- 腰围变化
- 饮食执行率
- 训练执行率
- 平均步数
- 平均睡眠
- 平均饥饿感
- 平均晨起精神
- 疼痛记录
- 本周体型照片

调整规则：

- 每周下降约 0.4～1.2 kg 且状态正常：维持计划。
- 连续两周下降不足 0.4 kg：建议每日减少约 150 kcal，或增加约 2000 步。
- 每周下降超过 1.2 kg 且有疲劳、头晕、饥饿或训练退步：建议增加 150～200 kcal 或降低训练量。
- 系统只给建议，修改计划必须由用户确认。

### 7.7 提醒

支持本地通知：

- 早晨称重
- 早餐、午餐、晚餐
- 锻炼
- 晚间记录
- 每周拍照
- 每周复盘

用户可开关、修改时间、区分工作日与周末并推迟提醒。

### 7.8 计划管理

计划字段：

- 名称
- 开始和结束日期
- 起始体重
- 阶段目标体重
- 长期目标体重
- 每日热量
- 每日蛋白质
- 每日饮水
- 状态

计划状态：未开始、进行中、已暂停、已完成、已放弃。

MVP 同一时间只允许一个进行中的计划。

#### 状态语义

- 未开始：计划已经创建，但用户尚未正式开始执行。
- 进行中：用户当前正在执行的唯一计划。
- 已暂停：用户主动暂时停止，保留以后恢复的可能。
- 已完成：用户主动确认计划结束；不得因为创建新计划而自动完成。
- 已放弃：用户明确决定不再继续该计划；不得由系统自动设置。

#### 状态转换原则

- 每一次状态改变都必须由用户主动发起并确认，系统不得根据日期、创建新计划或其他事件自动改变状态。
- 创建新计划时若已有进行中计划，必须先引导用户手动暂停、完成或放弃旧计划。
- 同一时间只允许一个进行中计划；恢复暂停计划前也必须先处理当前进行中的计划。
- 状态改变只影响计划生命周期，不得删除该计划的体重、围度、照片、饮食、锻炼或周报。
- 状态转换完成后应立即保存，并在历史计划中保持可追溯。

#### 仪式感与反馈动画

- 完成计划：二次确认后显示全屏庆祝动画，突出计划名称、执行天数、累计减重等成果，并给予明确、真诚的鼓励。动画结束前不得阻塞数据保存。
- 暂停计划：确认后使用平静、克制的过渡动画，说明记录已安全保留，可以随时恢复。
- 恢复计划：确认不存在其他进行中计划后，使用重新出发的正向动画。
- 放弃计划：先展示明确确认页，解释所有历史记录仍会保留；确认按钮使用“确认放弃计划”等完整文案，避免误触。完成后可使用低落但温和的动画，不得羞辱、责备或通过负罪感强迫用户继续。
- 动画应支持“减少动态效果”辅助功能；开启后以淡入、图标和文字反馈替代大幅移动、粒子或缩放。
- 所有结果动画均应提供明显的关闭按钮，不得强制观看完整动画。

#### 状态转换验收标准

1. 不存在任何无用户确认的计划状态改变。
2. 放弃、完成和切换进行中计划至少需要一次清晰确认。
3. 完成计划后展示庆祝反馈；放弃计划后展示温和的告别反馈。
4. 用户取消确认时，计划状态和数据均保持不变。
5. 状态改变后，旧计划及其全部历史数据仍可访问。
6. VoiceOver 能读出操作目的、风险和结果；减少动态效果开启时仍有完整反馈。

## 9. 计算规则

- BMI：当前体重除以身高米数的平方。
- 计划体重：MVP 使用起始体重到阶段目标之间的线性下降曲线。
- 7 日平均：使用当日及之前最多 7 条有效记录，缺失日不按 0 计算。
- 累计减重：起始体重减当前实际体重。
- 饮食执行率：完成 1 分，部分完成 0.5 分，其他 0 分。
- 训练执行率：完成 1 分，部分完成 0.5 分，未完成 0 分；计划休息日单独处理。

## 10. 隐私与安全

- 默认不注册账号。
- 数据优先保存在本地。
- 体型照片不得上传第三方分析服务。
- 开启 iCloud 前明确提示。
- 后台可模糊体型照片。
- P1 支持 Face ID。
- 删除计划和照片需要确认。

## 12. 非功能要求

- 首页快速显示本地数据。
- 核心操作最多三次点击完成。
- 无网络时核心功能正常。
- 照片列表使用缩略图，避免一次加载原图。
- 支持深色模式和 Dynamic Type。
- 修改未来计划不得影响已完成历史记录。

## 13. 技术建议

- SwiftUI + SwiftData
- Charts
- PhotosPicker
- UserNotifications
- LocalAuthentication
- 后续可接入 CloudKit 和 HealthKit

架构首版保持轻量，可采用 Feature 分层或 MVVM；在业务复杂度明显上升前，不引入过重抽象。

## 14. MVP 验收标准

1. 可以创建一个 8 周计划并生成 56 天数据。
2. 可以每日记录体重、腰围、睡眠和照片。
3. 可以查看每天四个餐次并标记状态。
4. 可以查看每日训练、目标步数并记录完成情况。
5. 可以显示计划、实际和 7 日平均体重趋势。
6. 可以自动汇总每周体重变化及饮食、训练执行率。
7. 重启 App 后数据不丢失。
8. 无照片权限或无网络时 App 不崩溃。
