# 体型变化追踪开发计划

## 1. 产品决策

用户真正需要的不是一个3D模型，而是可信地回答：

1. 我的身体是否正在发生变化？
2. 哪些部位发生了变化？
3. 当前减脂计划是否有效，值得继续坚持？

因此近期优先级为：

```text
标准化拍摄
→ 自动质量检查
→ 同角度自动对齐
→ 两日期照片/轮廓比较
→ 围度趋势估算
→ 参数化 3D 模型实验
```

完整3D人体扫描属于 P2 实验，不作为近期核心功能。手机扫描只能描述外形趋势，不能准确判断体脂、内脏脂肪或医学健康状态。

## 2. 用户价值

- 降低不同距离、角度和姿态造成的照片误差。
- 在体重短期不变时仍能观察腰腹等外形变化。
- 用叠加、轮廓和趋势表达变化，而不是要求用户凭记忆判断。
- 通过可信的阶段变化提高持续记录和减脂动力。
- 保证体型照片和衍生数据默认只保存在本机。

## 3. 标准化拍照需求

### 3.1 拍摄频率

- 默认建议每周一次，不要求每天拍摄。
- 用户仍可在任意日期补拍或替换。
- 提醒应强调相同时间、衣着和身体状态。

### 3.2 拍摄角度

首版必须支持：

- 正面
- 侧面
- 背面

后续可选增加左前45度和右前45度。每个角度必须独立保存状态，不得用文件名猜测角度。

### 3.3 拍摄前提示

- 手机保持竖直并固定在稳定位置。
- 全身从头顶到脚底进入画面。
- 相机高度尽量位于腰部到胸部之间。
- 使用相同距离、光线、背景和贴身衣物。
- 双脚位置、手臂姿态和视线保持一致。
- 正常呼吸，不刻意收腹。

### 3.4 相机取景引导

取景页应显示：

- 半透明人体轮廓参考线。
- 头顶、肩部、髋部和脚底参考区域。
- 当前目标角度。
- 简短姿势说明。
- 拍摄进度，例如“正面 1/3”。
- 取消、重新拍摄和使用照片操作。

拍摄引导不得遮挡身体关键区域。

### 3.5 自动质量检查

按阶段实现：

#### P0：基础检查

- 相机权限可用。
- 照片成功读取。
- 图片尺寸达到最低要求。
- 检测到一个主要人物。

#### P1：Vision 姿态检查

- 头部、双肩、双髋、膝盖和脚踝可见。
- 全身没有超出画面。
- 人体位于中央区域。
- 身体占画面高度处于合理范围。
- 正面/侧面姿态与目标角度基本一致。
- 失败时给出可操作提示，例如“请后退一步”或“脚部未完整入镜”。

#### P2：图像质量检查

- 判断过暗、过亮和明显模糊。
- 检测复杂背景或人物轮廓对比不足。
- 输出质量分数，低质量照片需要用户明确确认后才能保存。

### 3.6 自动拍摄

当人体连续稳定、完整入镜且姿态合格一段时间后，可以提供3秒倒计时自动拍摄。首版保留手动快门作为回退。

### 3.7 保存与隐私

- 原图压缩后保存到 App 私有目录。
- SwiftData 只保存随机文件标识和元数据。
- 保存拍摄角度、质量分数、图片尺寸、拍摄时间和引导版本。
- 替换或删除时清理旧文件。
- App 进入后台时隐藏照片。
- 默认不上传第三方服务，不将照片用于模型训练。

## 4. 照片对齐与变化展示

### 4.1 自动对齐

- 使用肩部、髋部和脚踝等人体关键点进行平移、缩放和轻微旋转校正。
- 对齐只生成展示副本，不覆盖原始照片。
- 对齐失败时允许退回未对齐比较。

### 4.2 对比方式

按优先级实现：

1. 左右并排。
2. 分割滑块前后对比。
3. 透明度滑块叠加。
4. 身体轮廓线叠加。
5. 同角度时间轴动画。

所有对比必须显示日期，并允许切换正面、侧面和背面。

## 5. 参数化 3D 实验

进入条件：

- 用户能够连续数周完成标准化拍摄。
- 照片对比功能被频繁使用。
- 用户反馈二维照片仍不足以判断变化。

候选路线：

- Vision 3D 人体姿态用于骨架和拍摄质量校验。
- 多角度人物轮廓拟合统一拓扑的人体模板。
- LiDAR 设备提供深度增强，但不作为基础功能依赖。
- 每日保存身形参数和关键围度，不优先保存大型独立网格。
- 使用同一模板参数插值生成动态变化，而不是直接混合不同扫描网格。

### 明确限制

- 不宣传毫米级或医学级精度。
- 不根据外形直接判断体脂率或内部健康状态。
- 围度和体积必须标记为估算，并提供质量或误差提示。
- 原始深度、点云和人体模型按照高度敏感健康数据处理。

## 6. 第一阶段验收标准

- 用户可以按正面、侧面和背面完成引导拍摄。
- 每一步明确显示当前角度和进度。
- 照片缺少完整人体时提供可理解的重拍提示。
- 用户可以跳过自动检查并手动确认，但 App 必须记录质量状态。
- 拍摄后照片进入现有私有存储、历史浏览和日期对比流程。
- 非 LiDAR iPhone 也能完成全部基础流程。
